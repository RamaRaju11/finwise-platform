?? Top 10 Critical Items (must-fix before launch)

  1. localStorage not persisted in 22+ modules � inputs reset on every page reload, making the tool feel broken. (Affects all groups)
  2. Division by zero in cashRunway � if monthly burn = 0 the runway calc returns Infinity/NaN and crashes the display. (cashRunway)
  3. loanRepaymentOptimizer dead simulate() function � old function still in code, confuses calculation path; remove and consolidate to simulateFull(). (loanRepaymentOptimizer)
  4. Payroll data not shared to ownersPay � owner pay module doesn't read employee/contractor payroll totals; users must double-enter. (ownersPay ? contractorPayroll)
  5. Invoice revenue not fed to budgetActualTracker � revenue figures must be re-entered manually; real data sits in invoiceTracker unused. (invoiceTracker ? budgetActualTracker)
  6. Tax estimate not wired to cashFlowCalendar � tax payment dates (Q1�Q4) should auto-populate cashFlowCalendar; currently manual. (taxEstimator ? cashFlowCalendar)
  7. grantFinder has no live data source � grant list is fully hardcoded; results will become stale without a refresh mechanism or API. (grantFinder)
  8. aiCfoChat has no conversation history � each message is independent; AI loses all prior context, giving incoherent multi-turn responses. (aiCfoChat)
  9. bankStatementUpload parses CSV only � OFX/QFX/QBO formats (QuickBooks, Mint exports) are unsupported; major adoption blocker. (bankStatementUpload)
  10. No global business profile data shared across modules � business name, industry, revenue tier entered repeatedly in every module. (All modules)

Cross-Cutting Themes

8 platform-wide issues

?? Data Persistence Gap

22+ modules do not save inputs to localStorage. Every reload requires full re-entry, making the platform feel unreliable and discouraging regular use.

Affects: All 30+ modules. Fix by adopting a consistent save/restore pattern using the existing fw_* key schema.

?? Missing Cross-Module Integration

Modules do not share data. Revenue entered in invoiceTracker, payroll in payrollEstimator, and taxes in taxEstimator are all siloed. Users re-enter the same numbers in every module.

Affects: budgetActualTracker, taxEstimator, ownersPay, cashRunway, monthlySnapshot, advisorDashboard, businessHealthScore, forecastPro.

?? No Export / Print / PDF

Nearly all modules lack a way to get data out. Small business owners need to share financial data with CPAs, lenders, and investors. The gap blocks real-world use.

Affects: 20+ modules. Minimum viable: browser window.print() with print-optimized CSS. Full solution: html2canvas or jsPDF.

?? Hardcoded IRS / Rate Constants

Tax brackets, SE tax rates, FICA rates, Section 179 limits, and mileage rates are embedded in logic with no annotation. They will become wrong in 2027 without visible flags.

Affects: taxEstimator, keyDeductions, payrollEstimator, contractorPayroll, ownersPay. Extract to a TAX_CONSTANTS_2026 config object.

?? Duplicate Amortization Logic

debtCommandCenter and loanRepaymentOptimizer both implement independent simulateFull() amortization functions. They will diverge over time and produce inconsistent results.

Affects: loanRepaymentOptimizer, debtCommandCenter. Extract to a shared amortize(principal, rate, term) utility function.

?? Timezone Date Handling

Date calculations using new Date('YYYY-MM-DD') shift by timezone offset in non-UTC browsers. This causes invoice due dates, tax deadlines, and calendar events to show the wrong day.

Affects: invoiceTracker, cashFlowCalendar, vendorBillTracker, taxEstimator. Fix: parse dates as new Date(yr, mo-1, day) or use UTC consistently.

??? No Global Business Profile

Business name, industry, entity type, and EIN are entered independently in every module. A global fw_profile localStorage object should be set once during onboarding and read everywhere.

Affects: All modules. Onboarding.html should be the single source of truth; modules read from fw_profile on load.

?? UI State Not Saved

Filter selections, active tabs, sort order, and toggled views reset on every page reload. Users must re-configure each module every visit, creating friction that erodes daily use habits.

Affects: loanMarketplaceEngine filters, budgetActualTracker category filter, bankStatementUpload view, grantFinder filter � and ~12 more modules.
